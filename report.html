YACC

Yacc is an LALR(1) parser generator. Yacc is a program that takes as input a specification of a syntax, and produces as output a procedure for recognizing that language. Historically, it is also called compiler-compilers.

I have used BISON in my project to parse the HTML using the CFG(Context free Grammar) that I have defined/written. Bison has several extensions over Yacc.
To install bison in linux, run the command
sudo apt-get install bison

Details:

  • Wrote Coherant productions to also resemble the AST of HTML.
  • Handled pointers carefully to avoid memory leak.

Abstract Syntax Tree

It is a tree representation of the abstract syntactic structure of source code written in a programming language. Each node of the tree denotes a construct occurring in the source code. The syntax is "abstract" in the sense that it does not represent every detail appearing in the real syntax, but rather just the structural or content-related details.



AST_NODE
  1. variable of NODE_TYPE : To store which type of tag this node represents. NODE_TYPE is a custom type ENUM to save space and order types of tags efficiently.
  2. variable of type string : To store the data of DATA NODES and other basic tags directly.
  3. vector of pair of string : To store attribute and their values.
  4. vector of AST_NODE pointers : To store children of AST nodes. Helps establish Parent-Child Relationship.
AST Functions
  • Create Node - To create a Node with the given Tag Type.
  • Add Attribute - From the children, add the attribute to current node.
  • Add Child - Given two nodes(par, kid), Add the kid to the parent.
  • Delete Tree - To free up dynamically assigned memory.
  • Mapper - Map type Data structure to map tags of HTML to commands of LaTeX
  • Fix LaTeX Chars - To find and fix the specials latex symbols that will not be displayed directly.
  • Create LaTeX - Recursively handles current node and visits all children of that node. This function traverses the AST in a Depth First manner.
